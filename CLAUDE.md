## Architecture

This is an **Expo 54 / React Native 0.81.5** app (New Architecture enabled) with file-based routing via `expo-router`.

### Layer Structure

```
app/          → Expo Router screens (file-based routing)
features/     → Domain-specific business logic (auth, cart, pdp, plp, search, account, …)
lib/          → Core utilities and API layer (Shopify GraphQL, storage, maps, OAuth)
ui/           → Reusable presentational components (primitives, layout, nav, product, …)
store/        → Global Zustand state (cartId, notifications, popup, prefs, wishlist)
```

**Dependency direction is one-way:** `app → features → lib`, `ui` is shared upward. UI components must not contain domain or business logic. Lower layers must not import from higher layers.

### Navigation

Root layout (`app/_layout.tsx`) wraps the app with `ShopifyAuthProvider`, `QueryClientProvider`, `GestureHandlerRootView`, `AuthGate`, `DrawerProvider`, `FontProvider`, and toast/popup hosts. Bottom tab navigation is driven by the custom `MenuBar` component. Key routes: `/home`, `/cart`, `/account`, `/search`, `/products/[handle]`, `/collections`, `/(auth)`, `/(onboarding)`.

### Data Fetching

- **Shopify Storefront GraphQL** via `graphql-request` — queries live in `lib/shopify/queries/`, types are auto-generated by `gql:gen` into `lib/shopify/gql/`
- **TanStack React Query** for caching (stale: 60s, cache: 5min, retry: 1, no refetch on focus/reconnect)
- Query key factory at `lib/shopify/queryKeys.ts`
- Service functions in `lib/shopify/services/` (products, cart, home, menus, brands, recommendations)
- **Shopify Customer Account API** (OpenID Connect OAuth) — token logic in `features/auth/`

### State Management

**Zustand** stores with MMKV persistence: `useCartId`, `useNotificationSettings`, `usePopupStore`, `usePrefs`, `useWishlist`.

### Styling

**NativeWind** (Tailwind CSS for React Native). Custom theme (colors, Geist font family, border radius) in `tailwind.config.js`. Prettier is configured with the Tailwind plugin (120 char width, no semicolons, trailing commas).

### Push Notifications

A separate **Cloudflare Worker** at `push-worker/` handles server-side push delivery. Configured via `push-worker/wrangler.toml`.

## Key Conventions (from AGENTS.md)

- **Functional components only** — no class components.
- **No `any`** unless it already exists in surrounding code and cannot be safely removed.
- All exported functions, components, and hooks must have **explicit TypeScript types**.
- New files live alongside their closest related feature — do not create new organizational schemes.
- Prefer **early returns** over deep nesting.
- Do not introduce commented-out code or TODOs unless explicitly requested.
- Before writing any code, verify that every import, API call, and config already exists in the codebase. **Do not invent APIs or env vars.**
- Never introduce new dependencies without explicit necessity.
- Keep diffs **minimal and localized** — avoid "while I'm here" changes, broad renames, or formatting churn.

## Environment Variables

All public env vars are prefixed `EXPO_PUBLIC_`. Key vars (see `.env.example` for full list):

| Variable | Purpose |
|---|---|
| `EXPO_PUBLIC_SHOPIFY_DOMAIN` | Shopify store domain |
| `EXPO_PUBLIC_SHOPIFY_STOREFRONT_TOKEN` | Storefront API token |
| `EXPO_PUBLIC_SHOPIFY_API_VERSION` | API version (e.g. `2025-07`) |
| `EXPO_PUBLIC_SHOPIFY_CUSTOMER_CLIENT_ID` | OAuth client ID |
| `EXPO_PUBLIC_GOOGLE_MAPS_IOS_KEY` / `ANDROID_KEY` | Native maps rendering |
| `EXPO_PUBLIC_GOOGLE_PLACES_KEY` | Places & Geocoding web service |
| `EXPO_PUBLIC_PUSH_WORKER_URL` | Cloudflare push worker endpoint |
